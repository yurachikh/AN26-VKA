# vulkan-air-monitor — суточная сводка по парку Vulkan Air

Раз в сутки скачивает трек по каждому борту парка с открытых зеркал
ADS-B Exchange (adsb.lol → airplanes.live → adsb.fi), разбирает на
отрезки и определяет аэропорты вылета/посадки.

По умолчанию мониторятся **UR-CQD, UR-CQE, UR-CQV, UR-CQZ** — текущий
парк Vulkan Air из 4 Ан-26. Регистрации автоматически резолвятся в
hex-коды через бесплатное API adsbdb.com и кэшируются в `fleet.json`.

## Пример вывода

```
=== Vulkan Air, 2026-05-25 UTC ===

UR-CQD  (Antonov An-26B, hex 50831c)
   05:42→08:14 UTC  EDDL Düsseldorf  →  LEMG Málaga  (152 мин)
   11:30→14:05 UTC  LEMG Málaga  →  EDDL Düsseldorf  (155 мин)

UR-CQE  (Antonov An-26B, hex 5083a3)
   06:12→09:18 UTC  LKKV Karlovy Vary  →  LHBP Budapest  (186 мин)

UR-CQV  (Antonov An-26B, hex 50831e)
   полётов не обнаружено (стоял на земле).

UR-CQZ  (Antonov An-26B, hex 50832a)
   данных ADS-B за день нет (не светил транспондер или вне покрытия).
```

## Быстрый старт локально

```bash
pip install airportsdata
python check_fleet.py                           # весь парк, вчера UTC
python check_fleet.py --date 2026-05-25
python check_fleet.py --fleet UR-CQE UR-CQV     # только часть
python check_fleet.py --fleet N628TS --title "SpaceX"   # любой другой парк
```

При первом запуске для новой регистрации скрипт сходит на adsbdb.com,
заберёт hex-код и тип ВС, и положит в `fleet.json`. Дальше — из кэша.

## Автоматизация через GitHub Actions

1. Создаёте репозиторий на GitHub, заливаете эти файлы:
   - `check_fleet.py`
   - `fleet.json`  (с предзаполненным UR-CQE; остальные доберутся при первом прогоне)
   - `.github/workflows/daily.yml`
2. На странице репозитория → **Actions** → разрешите workflow.
   Запускается ежедневно в 05:30 UTC (≈07:30 в Будапеште). Можно
   запустить вручную через **Run workflow** во вкладке Actions.
3. Отчёт сохраняется как артефакт билда (хранится 90 дней) и
   опционально отправляется в Telegram.
4. `fleet.json` коммитится обратно в репо при изменениях — это
   служит вашим журналом состава парка.

### Подключение Telegram

1. У [@BotFather](https://t.me/BotFather) создаёте бота → получаете
   токен вида `12345:AAH...`.
2. Пишете боту любое сообщение, потом открываете
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates` — там видно ваш
   `chat.id` (число).
3. В репозитории на GitHub: **Settings → Secrets and variables →
   Actions → New repository secret**, добавьте:
   - `TG_BOT_TOKEN` — токен бота
   - `TG_CHAT_ID` — ваш chat_id
4. С следующего запуска отчёт будет приходить в Telegram.

GitHub Actions для приватных репо даёт **2000 минут/месяц бесплатно**.
Этот job длится 20–40 секунд, лимит не выберете никогда.

## Локальный cron (если без GitHub)

```cron
30 7 * * *  cd /home/you/vulkan-air-monitor && /usr/bin/python3 check_fleet.py >> log.txt 2>&1
```

## Замечания и ограничения

- **UTC, не локальное время.** Перелёты через полночь UTC оказываются
  в разных файлах. Для Vulkan Air это редкость (Ан-26 — короткие
  плечи внутри Европы).
- **Аэропорт ищется в радиусе 15 км** от точки взлёта/посадки. Если
  посадка на необорудованной площадке или в зоне без покрытия — будут
  выведены сырые координаты.
- **«Стоял на земле»** означает: hex-код борта известен, но в этот
  день ни одна станция-фидер его не слышала. Возможны три причины:
  (а) борт реально не летал; (б) летал в зонах без покрытия (например,
  далеко в Африке); (в) транспондер был выключен (для гражданских
  чартеров крайне редко).
- **Скрипт не требует никаких API-ключей**: adsbdb.com и зеркала
  трекеров публично доступны для разумных нагрузок.
- **Парк меняется.** Если Vulkan Air получит новый борт или выведет
  старый — поправьте список `DEFAULT_FLEET` в начале `check_fleet.py`.

## Расширения

- **Diff с предыдущим днём** (новые маршруты, новые аэропорты в
  ротации) — несложно прикрутить, нужна только история отчётов.
- **KML/GPX-экспорт** маршрутов для просмотра в Google Earth.
- **Несколько компаний** — запускайте скрипт несколько раз с разными
  `--fleet` и `--title`, отчёты можно склеить в одно сообщение.
- **Email вместо Telegram** — добавляется одним шагом в workflow
  через `dawidd6/action-send-mail` или `smtplib`.
